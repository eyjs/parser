# Feedback 001 — 파싱 품질 이슈 수정

## 실제 PDF 테스트 결과 발견된 문제 (우선순위 순)

### 이슈 1: 줄바꿈 미병합 (심각도: 높음)
- **현상**: `"회 사"라 합니다)`, `같 습니다` — 단어 중간에서 PDF 줄 끊김이 그대로 유지됨
- **원인**: LineMerger가 PyMuPDF의 line 단위로만 병합 판단. 같은 block 내 연속 line이 단어 중간에서 끊긴 경우를 처리하지 못함
- **수정 방향**:
  1. 같은 block 내 연속 line은 기본 병합 (현재도 그래야 하는데 동작 안 함 — 디버그 필요)
  2. 줄 끝이 한글 자모/조사로 끝나고 다음 줄이 한글로 시작하면 공백 하나로 연결
  3. 줄 끝에 마침표/물음표/느낌표가 없으면 연속으로 간주
- **검증**: 제1조(목적) 본문이 한 문단으로 이어져야 함: `이 보험계약(이하 "계약"이라 합니다)은 보험계약자(이하 "계약자"라 합니다)와 보험회사(이하 "회사"라 합니다) 사이에...체결됩니다.`

### 이슈 2: 전체 페이지가 표로 잘못 추출 (심각도: 높음)
- **현상**: 보험약관 p11부터 동일 내용이 텍스트 블록 + 31열짜리 깨진 표로 이중 출력. 사업방법서는 전체가 31열 표
- **원인**: 이 PDF들은 페이지 전체에 테이블 라인(ruling lines)이 있는 레이아웃. pdfplumber가 이걸 표로 감지
- **수정 방향**:
  1. **전체페이지 표 감지**: 표의 bbox가 페이지 면적의 80% 이상이면 "레이아웃 표"로 판정
  2. **열 수 이상 감지**: cols >= 10이면 비정상 표로 판정 (보험 문서에서 10열 이상 표는 거의 없음)
  3. 레이아웃 표로 판정된 경우 → 표 추출 결과 버리고, 텍스트 블록만 사용
  4. `pdfplumber_tables.py`에서 필터링 추가
- **검증**: 사업방법서가 깨진 표 대신 일반 텍스트로 출력되어야 함

### 이슈 3: NBSP/특수공백 미정규화 (심각도: 중)
- **현상**: `이 보험계약` — non-breaking space(U+00A0)가 일반 공백이 아닌 특수문자로 남음
- **원인**: markdown_assembler 후처리에서 NBSP → 일반 공백 치환 누락
- **수정 방향**: `markdown_assembler.py`의 `_post_process()`에 NBSP 및 기타 유니코드 공백 정규화 추가
  - U+00A0 (NBSP), U+2002 (EN SPACE), U+2003 (EM SPACE), U+3000 (IDEOGRAPHIC SPACE) → 일반 공백
- **검증**: 출력 MD에 ` ` 등 비정상 공백이 없어야 함

### 이슈 4: 목차 페이지가 표로 추출 (심각도: 중)
- **현상**: 보험약관 목차 페이지(p2-p4)의 점선 패턴이 pdfplumber에 의해 표로 추출됨
- **원인**: NoiseDetector가 toc_pages=2로 감지했지만, 일부 목차 페이지는 필터링 안 됨. 또한 필터링된 페이지도 표 추출이 먼저 수행되어 결과에 포함됨
- **수정 방향**:
  1. 페이지가 NOISE로 분류되면 표 추출도 스킵하도록 파이프라인 순서 조정
  2. 표 내용에 점선(···, ····) 비율이 높으면 목차 표로 판정하여 제거
- **검증**: 목차 페이지가 출력 MD에 포함되지 않아야 함

### 이슈 5: 크로스페이지 표 병합 실패 (심각도: 중)
- **현상**: 상품요약서 p1의 표가 p2에서 25열짜리 깨진 표로 출력됨
- **원인**: 이슈2와 동일 — p2가 전체페이지 레이아웃 표로 인식됨
- **수정 방향**: 이슈2 수정으로 함께 해결될 것. 추가로 크로스페이지 표 병합 시 열 수가 이전 페이지와 극단적으로 다르면 병합 스킵
- **검증**: 상품요약서의 `보장명 | 보험금지급사유` 표가 정상 유지되고, 깨진 25열 표가 사라져야 함
